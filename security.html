1. The Copy-Mount Lifecycle

The Core "Prophylactic"

The Problem: Bind-mounts allow an AI to "see" and "edit" your host system in real-time. One malicious command can wipe your drive or leak your `.ssh` keys.

The Solution: We utilize an ephemeral Copy-Mount Sandbox. The Architect clones your repository into a temporary Docker volume. The AI works on the copy, not the source. If the agent attempts a "destructive" command, it only destroys a disposable mirror. Also, secrets (`.env`) are stripped before cloning.

3. Network Silence

Zero-Exfiltration

No Backdoors. No Leaks.
During the "Drafting" phase, the sandbox operates with --network none. The AI can write code and run local tests, but it is physically incapable of sending your data to an external server. Network access is only granted if you explicitly approve an npm install or curl during the Tier 3 Handshake.
